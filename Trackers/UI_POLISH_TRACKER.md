# DailyForge — UI Polish Tracker

Track UI issues as you spot them during testing. Fix in batches after each sprint.

---

## How to Use

1. **During testing:** Note any UI issue with screen name, description, and priority
2. **After sprint complete:** Review list, fix in one focused session
3. **Mark fixed:** Move to "Completed" section with date

**Priority Levels:**
- 🔴 **Critical** — Broken/unusable, blocks functionality
- 🟡 **Medium** — Looks wrong but works, should fix before launch
- 🟢 **Low** — Minor polish, nice-to-have

## Design Reference

**Correct colors (from React app):**
| Token | Hex | Usage |
|-------|-----|-------|
| Background | `#0c1222` | Page backgrounds |
| Card | `rgba(255,255,255,0.06)` | Card surfaces |
| Accent | `#D85A30` | Primary buttons, highlights |
| Green | `#1D9E75` | Success, positive actions |
| Gold | `#f59e0b` | PRs, streaks, warnings |
| Text Primary | `rgba(255,255,255,0.95)` | Main text |
| Text Secondary | `rgba(255,255,255,0.6)` | Subtitles |
| Text Muted | `rgba(255,255,255,0.4)` | Hints |
| Border | `rgba(255,255,255,0.06)` | Card borders |

**Button styles:**
- Primary: Filled with accent color (#D85A30)
- Secondary: Outline with white border
- Danger: Filled with red (#E53E3E)
- Ghost: Text only, no background

**Spacing:**
- Page padding: 16px horizontal, 20px top
- Card padding: 16px
- Between cards: 12px
- Between sections: 24px
