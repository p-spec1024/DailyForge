## Decision: Path A (strict per-level overlap, no clamp)

The diagnostic offered three paths:

- **A.** Spec eligibility formula is right; my hand-derived Appendix A was optimistic. Update the matrix.
- **B.** Eligibility formula too strict; widen to "any level overlaps + engine clamps to user level."
- **C.** Content gap; defer T3.5 until Sprint 13+ authors more techniques.

**Choice: A.**

### Why not B

Path B re-introduces exactly the dishonesty T3.5 was created to fix. T3 promised durations the content couldn't fill at the user's level; the engine quietly degraded. T3.5's whole point is `available` means "the user can do this length at their level, end of story." If we widen the formula to allow higher-level techniques with runtime clamping, an `available` bracket can return a duration shorter than the bracket's window because the user's level columns max out before the window does. That's the original bug in a new costume.

The `locked_by_level` state already handles this case correctly: "you can do this length, but not at your current level — unlocks at intermediate." That's a progression affordance, an honest invitation, not a deficiency. Hiding it under permissive `available` throws away the signal.

### Why not C

The 8 genuine empties (Pattern 1 in the diagnostic) are real content gaps and Sprint 13+ should author into them. But blocking T3.5 on content authoring would push the suggestion-engine MVP back weeks for cosmetic UX reasons. Empties are honest in v1: the bracket gets hidden in the picker, the user picks a different one, life goes on. New FUTURE_SCOPE entries below capture the gaps so they don't get forgotten.

### What this means for the spec

1. The eligibility formula in the v1 spec stays exactly as written.
2. Appendix A is replaced by §Corrected Appendix A below.
3. The defensive comment "*`empty` shouldn't happen given the seed*" in the v1 spec's `getAvailableDurations` contract was wrong. Empties happen and are valid. The helper's behavior is unchanged (it already returns `empty` correctly); only the comment is misleading.
4. Total smoke target changes: 60 generation cells → 40 generation cells. Revised total ≈ 1654 (1531 T2 baseline + 6 T3 throws + 75 matrix + 40 generation + 2 new throws + per-cell sub-assertions).

**Category:** Content / Approach 5

**Description:** Beginner energize users see only `0-10` and `10-20` brackets in the picker (plus `endless`). 21-30 and 30-45 are hidden because no beginner-tier energize technique reaches those durations at any level — they're not just locked, they're empty. Could overlap with #143's beginner-safe energizer authoring (Three-Part Breath with inhale emphasis, paced Cyclic Hyperventilation lite). Sprint 13+ content authoring.

**Surfaced from:** S12-T3.5 pre-flight diagnostic (Apr 29, 2026).

### #149 — Short-duration locked for intermediate/advanced focus and recover users

**Category:** Content / UX / Approach 5

**Description:** Intermediate and advanced focus users, and advanced recover users, see the `0-10` bracket as **empty**, not available. This happens because the higher-tier columns for these focuses are designed for longer protocols (Box Breathing intermediate = 10-15, etc.); the user *can* do shorter sessions but only at lower-tier columns the engine doesn't surface to them.

This is a structural feature of the current per-level duration model: progression at these focuses *removes* short-duration access. Two possible interpretations:

1. **Bug:** an advanced user should still be able to do a 5-min session — they just don't have a 5-min advanced *protocol*. Fix at the engine layer: allow lower-tier columns when no user-level columns fit a bracket. (This is Path B from the diagnostic, scoped to the empty case only.)
2. **Feature:** advanced users have outgrown 5-min sessions for these focuses; the picker correctly reflects that. UI shows "Higher tiers practice longer — try a longer session."

**Decision deferred** to Sprint 13 picker UX design. The current behavior (empty bracket = hidden in picker) is honest about the data.

**Surfaced from:** S12-T3.5 pre-flight diagnostic (Apr 29, 2026).

### #150 — Sleep techniques don't reach 25+ minutes at any level

**Category:** Content

**Description:** Sleep pool (Deep Sleep Induction, Sleep Preparation, 4-7-8 cluster) tops out at advanced ~20 min. The 30-45 bracket is empty for all three levels. The 21-30 bracket is empty at beginner and locked-by-advanced at intermediate. Fix: author or extend sleep techniques to reach 25-40 min — could be longer Yoga Nidra-style guided sequences or extended 4-7-8 advanced protocols. Sprint 13+ content authoring.
