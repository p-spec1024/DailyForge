## Why this ticket exists

T3 shipped a state-focus path with a **fixed-budget** contract: caller passes `time_budget_min`, engine fits content into it. The smoke run produced **24 DEGRADED cases out of 30** matrix entries — the engine returned durations different from the requested budget because state-focus content has rigid physiological min/max ranges that don't bend to arbitrary minute counts.

Examples from the T3 smoke:

| Case | Requested | Returned | Why |
|------|-----------|----------|-----|
| `energize / breathwork_tab / 30` | 30 min | 14 min | Morning Energizer beginner_max=8 |
| `recover / breathwork_tab / 3` | 3 min | 7 min | Diaphragmatic min=5 |
| `sleep / home / 60` | 60 min | 16 min | Deep Sleep Induction beginner_max=10 |

The engine reports honestly via `metadata.estimated_total_min`, but **the picker UX promises a duration the engine can't deliver.** That's the bug.

T3.5 fixes the contract, not the engine internals. The pick-and-fit logic, settle pool, role/standalone filters, and `durationsForLevel` carry forward. What changes:

1. **Picker contract:** caller passes a `bracket` (one of 5 values), not a minute count.
2. **Content-aware availability:** new helper `getAvailableDurations(focus_slug, breathwork_level)` returns which brackets the picker should expose for this user — UI never offers a bracket the engine would degrade on.
3. **Phase output names:** `settle / main / integrate` → `centering / practice / reflection` (user-facing language per Appendix A of the v2 spec).
4. **Endless mode:** 5th bracket, no fixed total — practice runs at the technique's natural max, with 2-min centering + 2-min reflection bookends.

This is a **refactor, not a rewrite.** T3 stays committed; T3.5 layers on top. Body focuses, mobility, full_body, recency, swap-counter, HTTP — all unchanged.

## Decisions locked (Apr 29, 2026 planning)

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| 1 | Dispatch contract | New `bracket` parameter (string enum). `time_budget_min` is ignored for state focuses. The HTTP layer (T7) validates `bracket` for state focuses and `time_budget_min` for body focuses. | Body and state focuses have different time semantics. Body answers "fill 30 min." State answers "give me an honest 10-20 min session." Overloading `time_budget_min` to carry both meanings with sentinel values would codify ambiguity and add a fallback path that has to be tested forever. |
| 2 | Endless centering/reflection defaults | Centering = 2 min, Reflection = 2 min | 3-3 imposes too much scripted scaffolding around what the user picked specifically because they want to just sit and breathe. 1-1 is below the floor — one minute isn't enough to anchor or integrate. 2-2 matches the spec note and is the defensible floor. FUTURE_SCOPE #146 personalizes from history later. |
| 3 | Smoke test scope | Full Appendix A matrix (75 cells) + a generation pass for every `available` cell, asserting `actual_total_min` falls within the bracket window. | T3.5 exists because T3's smoke proved degradation. If T3.5's smoke doesn't actually generate sessions in every available cell and check the duration honestly, the regression we're fixing can come back unnoticed. Cost is bounded (~30-40 generation calls). |
| 4 | Bracket set | `0-10`, `10-20`, `21-30`, `30-45`, `endless` — 5 values | Matches the v2 spec. 45-60 deferred to FUTURE_SCOPE #147 until content motivates it (currently no state-focus content reaches 60 min). |
| 5 | `getAvailableDurations` cell states | `available`, `locked_by_level`, `empty` — 3 states per (focus, level, bracket) cell | `available` = at least one main-eligible technique fits the bracket at this level. `locked_by_level` = a technique exists for this bracket but only at a higher difficulty (UI shows it greyed with a "Level up" affordance). `empty` = no technique exists for this bracket at any level (UI hides it). Distinction matters for FUTURE_SCOPE #143 (energize beginner) and progression UX. |
| 6 | Phase-name scope of rename | Engine output only. SQL column `breathwork_techniques.settle_eligible_for` keeps its name. The internal helper `pickSettleTechnique` keeps its name. | Renaming SQL columns and helpers would inflate the diff for zero functional reason. The rename is a user-facing concern; it lives in the JSON shape the engine returns. |
| 7 | Endless mode practice duration | Practice = picked technique's `<level>_duration_max`. No upper clamp. | Endless = "as long as the technique naturally allows." If a user picks endless and the engine selects Breath Counting at advanced (max=45), they get 45 min of practice. If they want shorter, they pick a numbered bracket. Endless is a deliberate user choice. |
| 8 | What happens if `bracket` is missing for a state focus | Throw `RangeError('state focus requires bracket parameter')` | Symmetric with T3's existing throws for body-focus-from-breathwork-tab and state-focus-from-body-only-tabs. Scope is locked; no silent fallthrough. |

## The 5 Brackets

| Bracket | Window (min) | Centering | Practice (clamped) | Reflection | Total |
|---------|-------------|-----------|--------------------|-----------|-------|
| `0-10` | 1–10 | 1 | clamped to fit (window − centering − reflection) within `[picked.<level>_duration_min, picked.<level>_duration_max]` | 1 | 1 + practice + 1 ≤ 10 |
| `10-20` | 10–20 | 2 | clamped to fit `[picked.<level>_duration_min, picked.<level>_duration_max]`, target = window − 2 − 2 | 2 | 2 + practice + 2, target ≤ 20 |
| `21-30` | 21–30 | 2 | clamped to fit, target = window − 2 − 2 (so 17–26) | 2 | 2 + practice + 2, target ≤ 30 |
| `30-45` | 31–45 | 3 | clamped to fit, target = window − 3 − 3 (so 25–39) | 3 | 3 + practice + 3, target ≤ 45 |
| `endless` | n/a | 2 | picked technique's `<level>_duration_max` (no clamp) | 2 | 2 + max + 2 |

**Practice clamp logic (numbered brackets only — endless skips the clamp):**

```
target_practice = window_max - centering - reflection
practice_minutes = CLAMP(
    target_practice,
    picked.<level>_duration_min,
    picked.<level>_duration_max
)
actual_total = centering + practice_minutes + reflection
```

For a bracket to be `available` for a (focus, level), there must exist at least one main-eligible technique at this level whose `[<level>_duration_min, <level>_duration_max]` range overlaps the practice window `[bracket_window_min - centering - reflection, bracket_window_max - centering - reflection]`. If the overlap is empty, the bracket is `locked_by_level` (the technique exists at higher difficulty) or `empty` (no technique fits at any level).

**Why the centering/reflection numbers vary by bracket:** a 0-10 session can't afford 2+2 of bookends — that leaves 6 minutes for practice and excludes any technique with min ≥ 7. 1+1 keeps the 0-10 window inclusive of techniques like Diaphragmatic (min=5). Larger brackets can afford more bookends and benefit from them.

## `getAvailableDurations` Contract

```js
/**
 * Returns the bracket availability for a (focus, level) pair.
 *
 * @param {string} focus_slug  - one of: energize, calm, focus, sleep, recover
 * @param {string} breathwork_level - one of: beginner, intermediate, advanced
 * @returns {Object} - shape:
 *   {
 *     focus_slug: 'calm',
 *     breathwork_level: 'beginner',
 *     brackets: [
 *       { id: '0-10',    state: 'available',       sample_count: 4 },
 *       { id: '10-20',   state: 'available',       sample_count: 6 },
 *       { id: '21-30',   state: 'available',       sample_count: 3 },
 *       { id: '30-45',   state: 'locked_by_level', sample_count: 0, unlocks_at: 'intermediate' },
 *       { id: 'endless', state: 'available',       sample_count: 6 }
 *     ]
 *   }
 *
 * - `state: 'available'` — at least one main-eligible technique at this level fits this bracket.
 * - `state: 'locked_by_level'` — at least one main-eligible technique fits this bracket at a HIGHER level.
 *     `unlocks_at` is the lowest level at which it becomes available (intermediate or advanced).
 * - `state: 'empty'` — no main-eligible technique fits this bracket at any level.
 *     UI hides the bracket entirely.
 *
 * `sample_count` is the number of techniques that fit at THIS level (used for UX copy like
 *   "6 calming techniques at this length"). For `locked_by_level` and `empty`, count = 0.
 *
 * For `endless`: a bracket is `available` if any main-eligible technique exists at the user's
 *   level. `endless` is `locked_by_level` if at least one exists at a higher level. `endless`
 *   is `empty` only if no main-eligible technique exists for this focus at any level (which
 *   shouldn't happen given the seed; defensive).
 */
function getAvailableDurations(focus_slug, breathwork_level)
```

**Implementation note:** the helper queries the same main pool that `loadStateMainPool` queries (the existing T3 helper), but groups results by bracket using each technique's `[<level>_duration_min, <level>_duration_max]` against the centering/reflection-adjusted practice window per bracket. The function returns synchronously over a single DB query (load all main-eligible techniques for the focus across all 3 levels, do the bracket bucketing in JS).

**Eligibility for "fits this bracket":**

```
For numbered bracket [bracket_min, bracket_max]:
  practice_window = [bracket_min - centering - reflection, bracket_max - centering - reflection]
  technique fits IF:
    technique.<level>_duration_min IS NOT NULL
    AND technique.<level>_duration_max IS NOT NULL
    AND [technique.<level>_duration_min, technique.<level>_duration_max] OVERLAPS practice_window
  (Overlap = max(a_min, b_min) <= min(a_max, b_max))
```

For the 0-10 bracket: practice_window = [-1, 8] → effectively [1, 8] (durations are positive).
For 10-20: practice_window = [6, 16].
For 21-30: practice_window = [17, 26].
For 30-45: practice_window = [25, 39].

For `endless`: technique fits IF `technique.<level>_duration_max IS NOT NULL`.

## `generateSession` Signature Update

Before T3.5:
```js
generateSession({
  user_id:         INT,
  focus_slug:      VARCHAR,
  entry_point:     ENUM('home', 'strength_tab', 'yoga_tab', 'breathwork_tab'),
  time_budget_min: INT NULLABLE
})
```

After T3.5:
```js
generateSession({
  user_id:         INT,
  focus_slug:      VARCHAR,
  entry_point:     ENUM('home', 'strength_tab', 'yoga_tab', 'breathwork_tab'),
  time_budget_min: INT NULLABLE,                                                 // body focuses
  bracket:         ENUM('0-10', '10-20', '21-30', '30-45', 'endless') NULLABLE  // state focuses (NEW)
})
```

**Validation rules (top of `generateSession`):**

1. If `focus_areas.type = 'body'` for the resolved focus AND `time_budget_min` is null → throw `RangeError('body focus requires time_budget_min')`. (Existing T2 behavior, unchanged.)
2. If `focus_areas.type = 'state'` AND `bracket` is null → throw `RangeError('state focus requires bracket parameter')`. (NEW.)
3. If `focus_areas.type = 'state'` AND `time_budget_min` is provided → silently ignore. (No throw — caller can pass it without breaking; engine won't use it.)
4. If `bracket` is provided but is not one of the 5 enum values → throw `RangeError('invalid bracket value: ${bracket}')`.
5. The existing throws for state-focus-from-body-only-tabs and body-focus-from-breathwork-tab carry forward unchanged.

## State-Focus Algorithm (T3.5 version)

```
generateStateFocus({ user_id, focus_slug, bracket }):

  1. RESOLVE LEVEL — only need breathwork_level (read user_pillar_levels).

  2. RESOLVE BRACKET CONFIG
     bracket_config = BRACKET_TABLE[bracket]
       // { window_min, window_max, centering, reflection, is_endless }

  3. CENTERING (no change vs T3 — rename only)
     pickSettleTechnique(focus_slug, breathwork_level, excluded)
       (existing T3 helper, unchanged)
     centering_minutes = bracket_config.centering

  4. PRACTICE
     IF bracket_config.is_endless:
       4a. Load main pool (existing loadStateMainPool helper)
       4b. Filter to techniques where <level>_duration_max IS NOT NULL
       4c. RANDOM_PICK
       4d. practice_minutes = picked.<level>_duration_max
       4e. No retry needed — endless always succeeds if pool is non-empty

     ELSE (numbered bracket):
       4a. Load main pool (existing loadStateMainPool helper)
       4b. Filter to techniques whose [<level>_duration_min, <level>_duration_max]
           overlaps the practice window
              [bracket_config.window_min - centering - reflection,
               bracket_config.window_max - centering - reflection]
       4c. IF filtered pool is empty → throw Error
              ('engine called with bracket that getAvailableDurations would mark
                locked_by_level or empty — caller must pre-check')
           This is a programming error, not a user error. UI should never call this.
       4d. RANDOM_PICK from filtered pool
       4e. target = bracket_config.window_max - centering - reflection
       4f. practice_minutes = CLAMP(target, picked.<level>_duration_min,
                                            picked.<level>_duration_max)
       4g. No retry loop needed — pre-filter guarantees fit

  5. REFLECTION
     reflection_minutes = bracket_config.reflection
     content_id = null (silent timer)

  6. RETURN {
       session_shape: 'state_focus',
       phases: [
         { phase: 'centering',  items: [centering_item] },
         { phase: 'practice',   items: [practice_item] },
         { phase: 'reflection', items: [reflection_item] }
       ],
       warnings: [],
       metadata: {
         estimated_total_min: centering_minutes + practice_minutes + reflection_minutes,
         bracket: bracket,
         is_endless: bracket_config.is_endless,
         user_levels: { ... }
       }
     }
```

**Invariant T3.5 must hold:** for every `bracket` value the engine accepts, if `getAvailableDurations(focus, level).brackets[bracket].state === 'available'`, then `generateStateFocus` produces a session whose `actual_total_min` lies within the bracket window (or matches `centering + technique_max + reflection` for endless).

The bounded retry loop and lowest-min fallback from T3 are **removed** for the numbered-bracket path — the pre-filter eliminates the failure mode they handled. The endless path needs no retry (it succeeds whenever any main-eligible technique exists at the level). This simplifies the function.

## Acceptance Criteria

T3.5 ships when **all of the following pass:**

1. **Schema:** No schema changes. `breathwork_techniques.settle_eligible_for` keeps its name; no new tables.
2. **Helper:** `getAvailableDurations(focus_slug, breathwork_level)` exists, returns the documented shape, runs in a single DB query.
3. **Engine:** `generateSession` accepts the new `bracket` parameter. State focuses route to the refactored `generateStateFocus` which uses bracket-driven allocation.
4. **Phase rename:** Engine output uses `centering / practice / reflection` for state-focus shape. Body-focus phase tokens unchanged.
5. **Throws:** State-focus-without-bracket throws `RangeError`. Invalid bracket value throws `RangeError`. T3's existing throws unchanged.
6. **Smoke matrix:** A new test pass `state-focus-bracket-matrix` produces 75 (focus, level, bracket) state assertions against Appendix A. All 75 match.
7. **Smoke generation:** For every cell where `state === 'available'` (60 cells), the test generates a session and asserts:
   - `phases.length === 3`
   - `phases[0].phase === 'centering'`
   - `phases[1].phase === 'practice'`
   - `phases[2].phase === 'reflection'`
   - For numbered brackets: `metadata.estimated_total_min ∈ [bracket.window_min, bracket.window_max]`
   - For endless: `metadata.estimated_total_min === centering(2) + picked.<level>_duration_max + reflection(2)` and `metadata.is_endless === true`
8. **Backward compatibility:** All T2 body-focus smoke (1531 assertions) still passes unchanged. T3's state-focus smoke is **replaced** (not extended) since the contract changed; T3's throw-based smoke for state-focus-from-body-only-tabs and body-focus-from-breathwork-tab carries forward unchanged.
9. **Total smoke target:** 1531 (T2 baseline) + 6 (T3 throw assertions, carried) + 75 (matrix) + 60 (generation) + per-case sub-assertions = **~1700+ pass, 0 fail**.
10. **`/review` grade:** A- or better. Architecture-pivot tickets get reviewed because the dispatch contract change has implications.

## Followups noted during T3.5 spec writing (eligible for FUTURE_SCOPE post-sprint-close)

1. **Deprecate `time_budget_min` for state focuses entirely.** Currently we silently ignore it. Once T7 ships and all callers route correctly, we could throw on its presence for state focuses to catch caller bugs. Sprint 13+ when call sites are stable.
